Introduction

AIGIT is an innovative AI-driven tool that enables users to interact with Git using natural language commands instead of predefined syntax. This eliminates the need for memorizing complex Git commands, making version control accessible to both beginners and professionals.

Features

  • Natural Language Processing: Converts user instructions into Git commands using AI.
  • Gemini API Integration: Leverages advanced AI models to interpret and execute commands.
  • Platform Independence: Runs on any device without requiring Python installation.
  • Terminal Automation: Can be invoked directly using the aigit keyword in the terminal.
  • Standalone Executable: AIGIT is available as an EXE file for easy deployment.

Installation & Setup

Prerequisites

  • Windows
  • Git installed on the system
  • Internet connection for AI processing

Installation Steps

Follow these steps to install and set up AIGIT on your Windows system:

Step 1: Download the AIGIT Executable

Visit the official AIGIT website.

Download the AIGIT executable file (.exe) to your system.

Step 2: Add AIGIT to the System Path

  • Identify the location where the AIGIT executable is downloaded or moved (e.g., C:\Program Files\AIGIT\aigit.exe).
  • Press Win + R, type sysdm.cpl, and press Enter.
  • Navigate to the Advanced tab and click on Environment Variables.
  • In the System Variables section, find and select Path, then click Edit.
  • Click New, then paste the path of the AIGIT executable directory (e.g., C:\Program Files\AIGIT\).
  • Click OK to save the changes and exit.

Step 3: Create a Shortcut Command for AIGIT

To use aigit as a command without typing the full path each time, follow these steps:

  • Open Command Prompt (Win + R, type cmd, and press Enter).
  • Run the following command to create a shortcut alias:
    doskey aigit=C:\Program Files\AIGIT\aigit.exe $*
  • This will allow you to run aigit from any terminal session.

Basic Commands

Natural Language Command Equivalent Git Command
aigit initialize a new repository git init
aigit clone the repository from GitHub <URL> git clone <URL>
aigit check the status of my repo git status
aigit add all changes git add .
aigit commit with message first commit git commit -m "Initial commit"
aigit push to GitHub git push origin main
aigit create a new branch called 'feature' git branch feature
aigit switch to branch 'feature' git checkout feature
aigit merge feature into main git merge feature
aigit pull latest changes git pull origin main

Technical Architecture

Workflow

  1. The user inputs a natural language command.
  2. AIGIT processes the input using the Gemini API.
  3. The AI interprets the command and maps it to an appropriate Git command.
  4. The system executes the Git command in the terminal.
  5. The user receives feedback on the operation.

Advanced Usage

  • Users can ask AIGIT to resolve merge conflicts automatically.
  • It supports multi-step commands, like "commit and push changes."
  • AIGIT can suggest best practices for repository management.

Key Components

  • AI Processing Module: Communicates with Gemini API to interpret commands.
  • Command Execution Module: Executes Git commands on the system.
  • Terminal Interface: Provides user interaction via command line.

Deployment

AIGIT is packaged as a standalone executable, allowing users to run it without installing Python. Future updates may include a web-based interface and mobile application.

Future Enhancements

  • Voice Commands: Enabling voice-controlled Git interactions.
  • GUI Integration: Providing a visual interface for ease of use.
  • Offline Mode: Implementing a local AI model for command processing without the internet.
  • Support for More Git Hosting Platforms: Expanding beyond GitHub to GitLab, Bitbucket, etc.
